# Stitch 设计框架（先设计后开发）

是的，建议先给 Stitch 一个明确的设计框架，再让它生成页面，然后我们按产出改代码。

## 1) 目标
- 统一视觉语言，减少页面各自发挥。
- 先产出页面结构和组件规范，再进入实现。

## 2) 适用范围（V2）
- 首页 / 记录 / 结算 / 设置
- 海报模块不在本轮范围内（后续 image2）

## 3) 设计约束（交给 Stitch）
- 风格关键词：简洁、轻拟物阴影、卡片化、信息优先。
- 配色：主色 #111111，强调色 #E0A800，中性色 #F5F6F8 / #E9EDF3。
- 圆角：卡片 20rpx，输入 12rpx，按钮 14rpx。
- 字号层级：标题 34rpx，正文 28rpx，辅助文案 24rpx。
- 交互优先级：主流程 CTA 永远突出（记一笔消费）。

## 4) Stitch 输出物要求
1. 页面线框（4页）
2. 组件清单（按钮、卡片、输入、标签、弹层、空状态）
3. 设计 token 表（颜色/圆角/阴影/间距/字号）
4. 页面状态覆盖（空状态、加载态、错误态）

## 5) 验收标准
- 4 个页面风格一致、组件可复用。
- 首页 30 秒主流程清晰可达。
- 与现有 `styles/stitch-theme.wxss` 可映射。

## 6) 给 Stitch 的提示词模板

```text
你是小程序产品设计师，请基于以下约束输出 V2 UI 方案：
- 业务：打工进度 + 轻记账（首页->记账->结算）
- 页面：首页、记录、结算、设置
- 不在范围：海报功能
- 风格：简洁、卡片化、轻阴影、信息优先
- 颜色：#111111 #E0A800 #F5F6F8 #E9EDF3
- 圆角：20/14/12
- 字号：34/28/24
请输出：
1) 4页线框说明
2) 组件清单与复用规则
3) token 表
4) 每页关键交互说明
5) 空/加载/错误三态说明
```
