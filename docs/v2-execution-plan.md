# 赚了么小程序 V2 执行方案（分步实施）

> 目标：避免“边做边想”导致阻塞，先方案后开发，按里程碑推进。

## 0. 工作原则

1. **先定义再开发**：每一阶段开始前必须有可评审文档。
2. **小步快跑**：每周只做 1~2 个核心目标，保证可验收。
3. **可回滚**：每个阶段均可独立上线，不依赖未完成模块。
4. **海报后置**：image2 海报能力单独立项，不影响主流程交付。

---

## 1. 总体节奏（6 个里程碑）

### M0：方案冻结（2~3 天）
**产出物**
- PRD V2（已建立，继续细化）
- 信息架构图（页面与跳转关系）
- 指标口径文档（收入、净收入、进度、时长）

**验收标准**
- 团队确认“V2 不做海报，仅保留占位入口”。
- 首页主流程定义达成一致。

### M1：设计系统与组件规范（3~4 天）
**产出物**
- Stitch 设计 Tokens（色板、字号、圆角、阴影、间距）
- 通用组件清单：按钮、卡片、标签、输入、弹层、空状态
- 页面线框（首页/记录/结算/我的）

**验收标准**
- 不再新增“临时样式”。
- 新页面默认复用组件库。

### M2：首页主链路重构（4~5 天）
**范围**
- 今日已赚、净收入、下班倒计时、状态切换、快速记账。

**验收标准**
- 30 秒内完成“打开首页 -> 记录消费 -> 切换状态”。
- 无阻塞错误，核心路径通过手测清单。

### M3：记录与结算页重构（4~5 天）
**范围**
- 记录页：消费列表、编辑/删除。
- 结算页：核心指标卡 + 时长结构。

**验收标准**
- 结算页数据与首页数据口径一致。
- 编辑/删除消费后全局数据正确刷新。

### M4：设置与隐私收敛（2~3 天）
**范围**
- 时薪、上下班、午休、隐私开关等配置统一。

**验收标准**
- 所有配置项生效路径清晰，重启后数据不丢失。

### M5：灰度与复盘（2~3 天）
**范围**
- 埋点核验、体验打磨、异常修复。

**验收标准**
- 核心漏斗数据可查看。
- 形成 V2 上线复盘文档。

---

## 2. 每阶段“先想清楚”的清单

每个阶段启动前，必须回答以下问题：
1. 这个阶段只解决哪 1~2 个问题？
2. 本阶段不做什么？（明确边界）
3. 验收口径是什么？如何证明做完了？
4. 失败风险是什么？回滚方案是什么？
5. 是否影响已有线上路径？

---

## 3. 任务拆分模板（建议）

每个开发任务必须包含：
- **背景**：为什么做
- **输入**：依赖的文档/接口/设计稿
- **输出**：具体交付文件
- **验收**：可验证结果
- **风险**：可能卡点与应对

> 推荐把任务粒度控制在 0.5~1.5 天内完成。

---

## 4. 海报模块（image2）后续立项方式

避免再次“主流程被海报卡住”，海报改为独立项目：
- 独立需求文档与里程碑
- 独立数据接口与素材依赖
- 不占用 V2 主流程迭代节奏
- 仅在结算页提供“敬请期待”入口
